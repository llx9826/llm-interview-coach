# 个性化定制

这一层用于把 skill 从“通用大模型面试教练”变成“围绕某个候选人的长期训练器”。

## 什么时候启用

以下情况优先读取本文件：

- 用户说“按我本人来”“更个性化”“结合我的项目”
- 用户贴了简历、项目经历、自我介绍
- 用户正在为特定岗位或特定公司准备
- 用户已经经过 1 轮以上 mock，需要根据历史问题继续训练

## 最低必要画像

先收集并整理：

1. 目标岗位
2. 目标公司类型
3. 工作年限
4. 最强的 2-3 个项目
5. 当前最弱的 3 个主题
6. 目标时间线
7. 中文面还是英文面
8. 用户自评强项和短板

如果用户信息不全，先用已有信息建立一个“临时画像”，不要停在收集环节太久。

## 候选人画像输出格式

输出一份简洁画像：

- 岗位类型
- 当前水平判断
- 最强项目
- 最可能被深挖的项目
- 最危险短板
- 最优先补的主题
- 面试风险等级

## 个性化优先级

问题优先级按这个顺序排：

1. 用户真实项目
2. 用户目标岗位的高频题
3. 用户最弱主题
4. 用户最可能被追问的系统/指标/失败案例
5. 通用基础题

不要反过来。不要让用户在还没准备好项目深挖时刷太多泛题。

## 个人短板地图

把短板归到这几类：

- 基础概念不稳
- 算法 tradeoff 弱
- 系统视角弱
- 指标意识弱
- 项目 ownership 弱
- 失败复盘弱
- 业务理解弱
- 英文表达弱

每次 mock 后，把暴露的问题挂回这张图里，优先补出现次数最多、面试杀伤力最大的项。

## 个性化题库构建

如果用户已提供项目背景，题库要改成：

- 40% 项目深挖
- 30% 岗位核心基础
- 20% 系统 / 指标 / serving
- 10% 扩展题

如果用户是转岗型候选人，额外多追：

- 为什么转这个方向
- 原有经历怎么迁移
- 为什么你能胜任这类岗位

## 个性化训练顺序

默认遵循二八法则：

1. 先补最容易挂的 20%：项目、指标、baseline、失败、tradeoff
2. 再补岗位核心基础：Transformer / 微调 / RAG / 推理优化
3. 最后才补扩展和边角题

## 长期训练原则

- 不要每次都从零开始
- 每次输出都要继承上次暴露的问题
- 把“用户最容易答虚的题”反复训练到稳定
- 优先追用户最真实、最有价值的项目经历

## 不足够个性化的表现

如果出现以下情况，说明 skill 没有真正个性化：

- 题目和用户简历几乎无关
- 项目追问停留在表层
- 不知道用户最弱主题是什么
- 每次 mock 像第一次见面
- 训练计划没有明显针对性
