# 诊断输出格式

每轮 mock、project review 或 answer review 结束后，用这个格式输出，形成训练闭环。

## 标准格式

1. `结论`
   这一题是 strong / passable / weak

2. `得分`
   1-10 分

3. `答对了什么`
   最多 3 点

4. `缺了什么`
   最多 3 点，必须具体

5. `面试官下一步会怎么追`
   给 1-3 个高概率追问

6. `更强版本`
   给一个更像面试高分回答的短版本

7. `暴露的薄弱点`
   从下面几类里选 1-3 个：
   - 基础概念
   - tradeoff
   - 指标意识
   - 项目 ownership
   - 系统视角
   - 业务视角
   - 失败复盘

8. `下一步训练动作`
   只给 1-3 个最小动作，不要给大而空建议

## 原则

- 反馈要短
- 反馈要能执行
- 反馈要基于本题暴露的问题
- 不要用太泛的安慰语
